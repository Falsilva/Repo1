Git - Controlador de Versão

Comandos básicos do Git

  • git status: mostra o estado do repositório e das alterações
  • git add <file>: prepara os arquivos para o versionamento, encaminhando-os para a INDEX, que é uma área temporária, antes da confirmação final, adicionando ou atualizando os arquivos que estão na Working Directory, que é a pasta de trabalho.
  • git commit -m "comentário": confirma as alterações no repositório local, os arquivos antes na área temporária, INDEX, no modo stage ou tracked, foram encaminhados para o HEAD, que é o repositório local.
  • git push: publica as alterações no repositório remoto, no serviço GitHub.
  • git pull: atualiza o repositório local trazendo todas as alterações do repositório remoto.

Outros comandos

  • git checkout -- <file>: desfaz as alterações da Working Directory, apagando todas as alterações feitas nos arquivos. ATENÇÃO, pois é o mesmo que utilizar o Ctrl+Z de tudo o que foi alterado.
  • git reset retira os arquivos da INDEX, tornando-os novamente Untracked, ou seja, não mapeados pelo Git.
  • git commit --amend: modifica a mensagem do commit mais recente, ou seja, o último commit feito por você no projeto. Além de você mudar a mensagem do commit, você consegue adicionar arquivos que você se esqueceu ou retirar arquivos comitados por engano. O git cria um commit totalmente novo e corrigido.
  • git checkout <nome_da_branch>: cria nova branch local ou alterna entre as branchs.
  • git checkout -b <nome_da_branch>: cria e seleciona a branch nova local.
  • git push --set-upstream origin <nome_da_branch>: cria a nova branch local no repositório remoto.
  • git branch -d <nome_da_branch>: deleta a branch.
